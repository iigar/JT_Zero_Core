 PROJECT OVERVIEW
═══════════════════════════════════════════════════════════════════════════

Project:    JT-Zero Runtime
Purpose:    Event/Reflex/Rule/Memory/Output engine for drone autonomy
Target:     Raspberry Pi Zero 2 W (compatible Pi 3/4/5)
Language:   C++17 (core), Python (API bridge), React (dashboard)
Arch:       Multi-threaded real-time runtime with 8 threads (T0-T7)

═══════════════════════════════════════════════════════════════════════════
 PHASE 1: ARCHITECTURE DESIGN
═══════════════════════════════════════════════════════════════════════════

- Defined 8-thread architecture:
  T0: System Supervisor (10Hz)
  T1: Sensor Pipeline (200Hz)
  T2: Event Engine (200Hz)
  T3: Reflex Engine (200Hz)
  T4: Rule Engine (20Hz)
  T5: MAVLink Interface (50Hz)
  T6: Camera Pipeline (15 FPS)
  T7: API Bridge

- Lock-free SPSC ring buffer (1024 events)
- Fixed memory pools, no dynamic allocation in hot paths
- System constraints: CPU <= 65%, RAM <= 300 MB

═══════════════════════════════════════════════════════════════════════════
 PHASE 3: CORE C++ RUNTIME
═══════════════════════════════════════════════════════════════════════════

Implemented 5 core engines:

1. Event Engine (event_engine.cpp)
   - SPSC ring buffer with atomic read/write indices
   - Event types: SENSOR, COMMAND, SYSTEM, REFLEX, RULE, WARNING, ALERT
   - Priority levels: LOW, NORMAL, HIGH, CRITICAL
   - Zero-allocation in publish path

2. Reflex Engine (reflex_engine.cpp)
   - Sub-microsecond response time (~1.2 us)
   - Built-in reflexes:
     * Emergency Stop (altitude > 150m)
     * Low Battery (< 11.1V)
     * Altitude Limit (> 120m)
   - Custom reflex registration via function pointers

3. Rule Engine (rule_engine.cpp)
   - Complex condition evaluation (20Hz)
   - Built-in rules:
     * Auto-RTL on low battery (< 11.5V + 120s flight)
     * GPS-lost position hold
     * Takeoff completion detection (alt > 2m)
   - Evaluation counter per rule

4. Memory Engine (memory_engine.cpp)
   - Telemetry history ring buffer (2048 records)
   - Event history ring buffer (512 events)
   - No dynamic allocation, fixed-size pools

5. Output Engine (output_engine.cpp)
   - Action types: GPIO, API_CALL, LOG, COMMAND
   - Thread-safe action queue
   - Execution statistics tracking

═══════════════════════════════════════════════════════════════════════════
 PHASE 4: SENSOR MODULES
═══════════════════════════════════════════════════════════════════════════

Implemented sensor simulation (sensors.cpp):
- IMU:           200Hz (gyroscope + accelerometer)
- Barometer:     50Hz  (pressure + altitude + temperature)
- GPS:           10Hz  (lat/lon/alt/speed + fix quality)
- Rangefinder:   50Hz  (distance + quality)
- Optical Flow:  50Hz  (flow_x/y + quality)

All sensors generate realistic simulated data with noise.

═══════════════════════════════════════════════════════════════════════════
 PHASE 5: CAMERA PIPELINE
═══════════════════════════════════════════════════════════════════════════

Implemented (camera_pipeline.cpp):
- FAST-9 corner detector
- Lucas-Kanade sparse optical flow tracker
- Visual Odometry estimator
- Resolution: 320x240 @ 15 FPS
- Simulated camera generates moving test patterns
- VO outputs: delta_x, delta_y, delta_yaw, quality, feature count

Implemented (mavlink_interface.cpp):

- Simulated Flight Controller: ArduPilot 4.5.0, QUADROTOR

 PHASE 7: PYTHON BINDINGS (pybind11)
═══════════════════════════════════════════════════════════════════════════

Exposed full C++ runtime to Python (python_bindings.cpp):
  - jtzero_native.JTZeroRuntime class
